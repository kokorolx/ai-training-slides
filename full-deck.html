Trước khi bắt đầu
Thuật ngữ quan trọng
AI Session
Một lần bạn mở AI assistant và trò chuyện. Khi đóng tab → session kết thúc, AI quên hết.
Context Window
Bộ nhớ tạm của AI trong 1 session. Giới hạn (~200k tokens). Khi đầy → AI bắt đầu quên.
Slash Command
Lệnh bắt đầu bằng / (ví dụ: /pr-code-reviewer). Gọi skill đã viết sẵn, AI thực hiện đúng quy trình.
Token
Đơn vị đo bộ nhớ AI. ~200k tokens ≈ 150k từ tiếng Anh ≈ 75k từ tiếng Việt.

# CLAUDE.md
## Tech Stack
- Nuxt 3 + TypeScript
- Express :3000 (user-facing API)
- NestJS (pricing-manager)
## Conventions
- 2 spaces, single quotes, no semicolons
- Always brace control statements
## Anti-patterns
- Never use window.localStorage directly
- No <v-icon> — use Font Awesome <i> tags
- No <p> tags for UI text, use <div>/<span>
Key Insight
File này là "bộ nhớ dài hạn" của AI về project. Thay đổi codebase → update ngay.

Block 2 — Skills
/systematic-
debugging
Dùng khi gặp bất kỳ bug hay unexpected behavior. Evidence trước hypothesis.
AI Red Flags
— Fix without finding root cause
— "It might be..." (assumption without evidence)
— Add try/catch to hide the error
1
Reproduce
Minimal reproducible case. Tái hiện được 100% mới bắt đầu debug.
2
Isolate
Tìm layer có bug: network / logic / state / DB. Loại trừ từng phần.
3
Hypothesize
Nêu hypothesis cụ thể + cách test nó. "If X then Y" — falsifiable statement.
4
Fix + Verify
Fix root cause, không symptom. Verify với evidence — test pass, curl output, screenshot.

Block 3 — Spec-Driven
Spec là gì —
và AC là gì
SPEC = WHAT & WHY
Không phải HOW
Bad
"Dùng Redis cache TTL 5 phút, invalidate khi update"
Good
"Profile load < 200ms. Data consistent sau update."
Rule
Spec định nghĩa outcome, không implementation. AI tự quyết định HOW.
AC = CÂU HỎI YES/NO
Verifiable, Specific
Bad
"User có thể login bình thường"
Good
"POST /login với valid creds → 200 + JWT trong 500ms"
Feature done = tất cả AC = Yes
Không phải khi dev tự cảm thấy xong
